## ISSUE: BlackoutWindow class never unregistered

- **Location**: BlackoutWindow.cs:55-76, Program.cs:92
- **Severity**: Medium

`ScreenNap_Blackout` window class is registered via `RegisterClassExW` but never unregistered at shutdown. Program.cs:92 only unregisters `MessageWindowClassName`.

**Fix**: Add `UnregisterClassW("ScreenNap_Blackout", hInstance)` to the cleanup path.

---

## ISSUE: Shell_NotifyIcon return value not checked

- **Location**: TrayIcon.cs:31-32
- **Severity**: Medium

`Shell_NotifyIconW(NIM_ADD, ...)` return value is ignored. `_created` is set to `true` regardless of success. Subsequent `NIM_MODIFY` / `NIM_DELETE` calls would operate on a non-existent icon.

**Fix**: Check return value before setting `_created = true`.

---

## ISSUE: Exit path cleanup incomplete

- **Location**: ContextMenu.cs:62-66, Program.cs:89-92
- **Severity**: High

Exit flow:

1. `ContextMenu.HandleCommand` calls `ReleaseAll()` then `PostQuitMessage(0)` (ContextMenu.cs:64-65)
2. Message loop exits, cleanup runs (Program.cs:89-92)

Problems:

- `DestroyWindow(s_messageWindow)` is never called. The WM_DESTROY handler (Program.cs:112) never fires during normal exit.
- `UnregisterClassW` is called while the message window is still alive — may fail.
- `ReleaseAll()` is called twice (once in HandleCommand, once in cleanup). Safe but redundant.

**Fix**: Call `DestroyWindow(s_messageWindow)` before cleanup, or restructure so that Exit sends WM_CLOSE/WM_DESTROY to the message window and cleanup runs from the WM_DESTROY handler. Remove redundant `ReleaseAll()`.

## ISSUE: SetTimer return value not checked

- **Location**: BlackoutWindow.cs:43
- **Severity**: Low

If `SetTimer` fails, the topmost maintenance timer silently doesn't work. The blackout window could be covered by other windows.

**Fix**: Check return value (0 indicates failure).

## DESIGN: Right-click dismiss not in spec

- **Location**: BlackoutWindow.cs:123-126
- **Decision needed**: Keep or remove?

The spec (SPEC.md) specifies double-click only for dismiss. The implementation also handles `WM_RBUTTONUP` as a safety measure for when the main monitor is blacked out. The tooltip still says "Double-click to dismiss" with no mention of right-click.

**Recommendation**: Keep right-click dismiss (reasonable safety feature). Update SPEC.md and tooltip text to reflect both methods.
